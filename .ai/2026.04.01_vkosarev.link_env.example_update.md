# 2026-04-01 — Обновление .env.example для vkosarev.link

## Задача
Обновить файл `.env.example` в `vkosarev.link`, чтобы он отражал все реальные переменные окружения, необходимые для работы сайта и сервисов.

## Проблема (было)
Старый файл `.env.example` содержал только:
- `CERT_DIR` с неправильным путём (`/tools/nginx/cert` вместо `/root/cert`)
- `PORTAINER_DATA` (закомментирована)
- `OPENVPN_DATA` (закомментирована)
- `FRPS_CONFIG` (закомментирована)
- `PROMETHEUS_DATA` (закомментирована)
- `NODE_NAME`

**Отсутствовали переменные для 3x-ui-exporter:**
- `XUI_EXPORTER_ORIGIN`
- `XUI_EXPORTER_USERNAME`
- `XUI_EXPORTER_PASSWORD`
- `XUI_EXPORTER_PORT`

## Решение (стало)
Полностью переработан файл `.env.example` с:

### ✅ Добавлено:
1. **Переменные 3x-ui-exporter** (для сбора метрик Xray через Prometheus)
   - `XUI_EXPORTER_ORIGIN` — адрес панели 3x-ui внутри Docker-сети
   - `XUI_EXPORTER_USERNAME`, `XUI_EXPORTER_PASSWORD` — учётные данные
   - `XUI_EXPORTER_PORT` — порт слушания экспортёра (3001)

2. **Лучшая организация:**
   - Разделены на секции с заголовками (TLS-сертификаты, 3x-ui, данные контейнеров, инфраструктура)
   - Добавлены подробные комментарии для каждой переменной
   - Указаны значения по умолчанию, когда применимо

3. **Исправлены пути:**
   - `CERT_DIR=/root/cert` (вместо `/tools/nginx/cert`)

## Переменные в docker-compose.yml (найдено через grep)

| Переменная | Где используется | Значение по умолчанию |
|---|---|---|

| `OPENVPN_DATA` | openvpn volumes (закомментирован) | `./openvpn/data` |
| `PROMETHEUS_DATA` | prometheus volumes | `./prometheus/data` |
| `NODE_NAME` | prometheus environment | `vkosarev.link` |

## Файл обновлён
✅ `vkosarev.link/.env.example` — готов к использованию

## Рекомендации
1. При первом запуске сервисов убедитесь, что скопировали `.env.example` → `.env` и заполнили реальные значения
2. Особенно важны:
   - Пароль `XUI_EXPORTER_PASSWORD` — заменить на реальный
   - `CERT_DIR` — убедиться, что путь содержит `fullchain.pem` и ключ
   - `NODE_NAME` — совпадает с доменом/именем хоста
3. Пароль Grafana **не требуется** для vkosarev.link (используется Prometheus только)
