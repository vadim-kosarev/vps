## Grafana: что брать для Docker

- data/grafana.db — база данных (обязательно)
- conf/ — все конфиги, особенно provisioning/
- data/plugins/ — все плагины

**Не требуется:**
- data/log/, data/pdf/, data/png/, data/csv — только для отладки/экспорта
- bin/, public/, tools/, docs/, packaging/, storybook/, npm-artifacts/ — не нужны для контейнера

**Пример volume-маппинга:**

```yaml
  grafana:
    image: grafana/grafana:10.4.2
    volumes:
      - ./grafana/data/grafana.db:/var/lib/grafana/grafana.db
      - ./grafana/conf:/etc/grafana
      - ./grafana/data/plugins:/var/lib/grafana/plugins
    environment:
      - GF_SECURITY_ADMIN_PASSWORD=yourpassword
    ports:
      - "3000:3000"
    restart: unless-stopped
```

**Действия:**
- Скопировать нужные каталоги и файлы в luigi/grafana/
- Проверить работоспособность в Docker Compose
- Обновить отчёт по итогам миграции
