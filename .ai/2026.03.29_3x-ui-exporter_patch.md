# 2026.03.29_3x-ui-exporter_patch.md

## Описание проблемы

На сервере vkosarev.link 3x-ui-exporter не работал корректно. При обращении к метрикам:

```
curl http://localhost:3001/metrics
```

возвращался ответ:

```
{"statusCode":500,"error":"Internal Server Error","message":"Cannot read properties of null (reading 'includes')"}
```

Ошибка возникала из-за строки в файле `/app/build/xui.js`:

```js
onlinesResp.obj.includes(e.email)
```

Если поле `onlinesResp.obj` отсутствует, возникала ошибка выполнения.

## Решение

Был применён следующий патч внутри контейнера 3x-ui-exporter:

```sh
sed -i "s/onlinesResp.obj.includes(e.email)/onlinesResp.obj ? onlinesResp.obj.includes(e.email) : false/g" /app/build/xui.js
```

Теперь строка выглядит так:

```js
onlinesResp.obj ? onlinesResp.obj.includes(e.email) : false
```

Это устраняет ошибку при отсутствии поля `obj` в ответе onlinesResp.

## Команда для применения патча

Выполнить на сервере:

```
docker exec 3x-ui-exporter sh -c '
  sed -i "s/onlinesResp.obj.includes(e.email)/onlinesResp.obj ? onlinesResp.obj.includes(e.email) : false/g" /app/build/xui.js
  echo "✅ Патч применён"
'
```

## Статус

Патч протестирован, экспортер работает корректно на vkosarev.link.
