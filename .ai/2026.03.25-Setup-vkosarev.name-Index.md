### Сервис в docker-compose.yml (итоговый вариант)

```yaml
nginx:
  image: nginx:1.27-alpine
  container_name: nginx
  restart: unless-stopped
  network_mode: host          # ← ключевое решение (см. раздел 4)
  volumes:
    - ./nginx/conf.d:/etc/nginx/conf.d:ro
    - ./nginx/www:/etc/nginx/www:ro
    - ./nginx/.htpasswd:/etc/nginx/.htpasswd:ro
    - ${CERT_DIR:-/root/cert}:/root/cert:ro
```

### Адаптации конфига относительно хостового nginx

| `root /etc/nginx/www/` (502, **не существовал**) | `root /etc/nginx/www/` (**работает**) | volume-монтирование |

После первого деплоя все frps-проксированные сервисы возвращали `502`.

**Диагностика:**
```
ss -tlnp | grep -E ':(980|2283|4040|5000)'
# → 127.0.0.1:2283, 127.0.0.1:4040, 127.0.0.1:5000, 127.0.0.1:980
```

Из `frps/frps.toml`:
```toml
proxyBindAddr = "127.0.0.1"
```

## 4. Корень проблемы и решение

### Проблема

Первая версия docker-compose.yml использовала:
```yaml
extra_hosts:
  - "host.docker.internal:host-gateway"
```
и `proxy_pass http://host.docker.internal:2283`.

`host.docker.internal` резолвится в IP Docker bridge gateway
(`172.17.0.1` или аналог). Но frps слушает на `127.0.0.1` (loopback).
Из Docker-сети `127.0.0.1` хоста **недоступен**.

### Решение: `network_mode: host`

Аналогично тому, как уже работает контейнер `frps`.
При `network_mode: host` сетевой стек контейнера = сетевой стек хоста,
`localhost` = `127.0.0.1` хоста.

**Изменения:**
1. `docker-compose.yml` — убраны `ports:` и `extra_hosts:`, добавлен `network_mode: host`
2. `nginx/conf.d/vkosarev.name.conf` — заменено `host.docker.internal` → `localhost`

### Итог после исправления

| Порт | Результат |
|------|-----------|
| `80` | ✅ `301 → https://vkosarev.name:1443/` |
| `1443` | ✅ `200` статический сайт |
| `7601` IMMICH | ✅ `200` HTML приложения |
| `3181` | ⚠️ `502` — туннель `:3180` не зарегистрирован в frps (ожидаемо) |
| `5001` Frigate | ✅ `401` без auth / `200` с `admin:noname564` |
| `981` HIKCAM | ✅ `401` без auth / `200` с `admin:noname564` |
| `4041` Subsonic | ✅ `302` редирект на логин |

**Сертификат:** действителен до `2026-06-13`.
