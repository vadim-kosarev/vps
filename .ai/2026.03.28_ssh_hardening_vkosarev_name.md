# 2026.03.28_ssh_hardening_vkosarev_name.md

**Дата:** 2026-03-28
**Сервер:** vkosarev.name

## Цель
Ограничить SSH-доступ для пользователя root только по сертификату (SSH-ключу), полностью запретить вход по паролю.

## Действия
1. Проверена текущая конфигурация `/etc/ssh/sshd_config`:
    - Было: `PermitRootLogin yes`, `PasswordAuthentication yes`, `PubkeyAuthentication yes`.
    - Это означало, что root мог входить по паролю и по ключу.
2. Создана резервная копия файла: `/etc/ssh/sshd_config.bak`.
3. Внесены изменения:
    - `PasswordAuthentication no` — запрещён вход по паролю для всех пользователей.
    - `PermitRootLogin prohibit-password` — root может входить только по ключу.
4. Перезапущена служба SSH для применения настроек.
5. Проверено, что изменения применились:
    - Итог: `PermitRootLogin prohibit-password`, `PasswordAuthentication no`.

## Результат
- Вход по паролю для root и других пользователей отключён.
- Вход по SSH-ключу для root разрешён.
- Безопасность SSH усилена согласно требованиям.
- Резервная копия старого конфига: `/etc/ssh/sshd_config.bak`.
