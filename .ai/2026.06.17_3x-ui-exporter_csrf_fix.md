# 2026.06.17 — 3x-ui-exporter: fix CSRF compatibility

## Проблема

Grafana на vkosarev.name перестала получать данные от 3x-ui exporter.

**Причина:** образ `m4l3vich/3x-ui-prometheus-exporter:latest` был пересобран 2026-06-15 и теперь требует endpoint `/csrf-token` для авторизации. Панели 3x-ui на всех серверах работают на v2.8.11, в которой этого endpoint нет (появился в v3.x).

Exporter не имеет версионных тегов — только `latest`, поэтому любой пересборка образа может сломать совместимость.

**Ошибка в логах:**
```
Scrape failed: Login failed FetchError: [GET] "https://vkosarev.name:2055/vkosarev.name.eu/csrf-token": 404 Not Found
```

## Решение

Создан entrypoint-wrapper `shared/3x-ui-exporter/patch-entrypoint.sh`, который при каждом старте контейнера патчит `/app/build/xui.js` — оборачивает вызов `fetchCsrfToken()` в try/catch, чтобы при 404 exporter продолжал работу без CSRF-токена.

### Изменения в docker-compose (все 3 сервера):
```yaml
3x-ui-exporter:
  entrypoint: ["/bin/sh", "/patch-entrypoint.sh"]
  command: ["pnpm", "start"]   # CMD сбрасывается при override entrypoint
  volumes:
    - ../shared/3x-ui-exporter/patch-entrypoint.sh:/patch-entrypoint.sh:ro
```

### Нюанс с Docker Compose
При переопределении `entrypoint` Docker Compose сбрасывает `CMD` образа в null. Без явного `command: ["pnpm", "start"]` контейнер стартует и сразу завершается с кодом 0.

### Диагностика патча (3 ветки логирования):
- `patched fetchCsrfToken() fallback` — патч применен
- `already patched, skipping` — повторный старт, патч уже есть
- `WARNING: fetchCsrfToken() pattern not found` — код exporter изменился, нужна ревизия патча

## Совместимость при обновлении

- **3x-ui обновлен до v3.x:** `/csrf-token` доступен, `fetchCsrfToken()` отрабатывает штатно, catch не срабатывает. Патч безвреден.
- **Exporter изменил код:** sed не найдет паттерн, выведет WARNING. Если 3x-ui на v3.x — работает нативно. Если на v2.x — нужно обновить патч.
- **Удаление патча:** после обновления 3x-ui до v3.x на всех серверах можно убрать `entrypoint`, `command` и volume из docker-compose.
