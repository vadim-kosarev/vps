# Local: Network Routing Fix & Tools Setup

**Date:** 2026-07-07

## 1. Home Network Routing (192.168.2.0/24 → 192.168.1.x)

### Topology

```
Internet
   |
Router 1 — Keenetic Viva (192.168.1.1)
   |         LAN: 192.168.1.0/24
   |         WAN: ISP via PPPoE (NEWCOMM)
   |
   +-- [Wi-Fi 5 GHz] --> Router 2 — Keenetic Extra
                          WAN: 192.168.1.91
                          LAN: 192.168.2.0/24 (gateway: 192.168.2.1)
```

### Problem

Devices on 192.168.1.x could not reach the 192.168.2.0/24 subnet despite a static route being configured on Router 1.

### Root Cause

Router 2 (Keenetic Extra) blocked all inbound traffic arriving on its WAN interface (WISP 5 GHz) by default. The static route on Router 1 was correct, but Router 2's firewall silently dropped packets from 192.168.1.x before they reached the LAN.

### Fix Applied

**Router 1 (192.168.1.1)** — static route (was already configured):
- Network Rules → Routing → User-Defined Routes
- Destination: `192.168.2.0/24`, Gateway: `192.168.1.91`, Add automatically: Yes

**Router 2 (192.168.1.91)** — firewall rule added:
- Network Rules → Firewall → tab: WISP 5 GHz (LECHAYA-21-K5G)
- Rule: Action=Permit, Protocol=IP, Source IP=`192.168.1.0/24`, Destination IP=Any
- Name: "Allow 192.168.1.x", Enabled: Yes

### Result

```
ping 192.168.2.1
Reply from 192.168.2.1: bytes=32 time=2ms TTL=64  (x4, 0% loss)

Test-NetConnection 192.168.2.1 -Port 80 → True
```

All devices on 192.168.1.x can now reach 192.168.2.0/24 without per-host route configuration.
