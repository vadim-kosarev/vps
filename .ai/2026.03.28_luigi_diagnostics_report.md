# Диагностика сервисов и экспортеров на Windows-машине luigi (2026-03-28)

## Цель
Провести аудит и документирование всех сервисов, экспортеров и утилит, запущенных на рабочей станции luigi, для последующего описания инфраструктуры и мониторинга.

## Ход диагностики

1. **Удалённое подключение**
   - Был настроен PowerShell Remoting с помощью отдельного пользователя `local-admin`.
   - Проверена возможность входа через `Enter-PSSession -ComputerName luigi -Credential luigi\local-admin`.

2. **Аудит процессов и служб**
   - Получен список всех запущенных процессов через `Get-Process | Sort-Object ProcessName | Select-Object ProcessName, Id, Path`.
   - Получен список всех запущенных служб через `Get-Service | Where-Object {$_.Status -eq "Running"}`.
   - Проверены процессы, связанные с мониторингом, экспортерами, мультимедиа и сетевыми сервисами.

3. **Проверка Docker и WSL**
   - Docker Desktop установлен, но не запущен (ошибка запуска).
   - WSL-дистрибутивы отсутствуют.

4. **Проверка портов**
   - Выполнена команда `netstat -ano | findstr LISTEN` для выявления слушающих портов и их PID.
   - Сопоставлены PID с процессами и службами.

5. **Поиск специфических экспортеров**
   - Обнаружены службы: `IPerfExporter`, `InternetExporter`, `windows_exporter`, `Prometheus`, `Grafana`, `Subsonic`, `qBittorrent` и др.
   - Все экспортеры и мониторинговые сервисы работают как Windows-службы (часть через NSSM).

## Выводы

- На luigi развернут полноценный стек мониторинга: Prometheus, Grafana, windows_exporter, iperf_exporter, InternetExporter.
- Для проверки скорости интернета используется iperf_exporter (порт 9115), интегрированный с Prometheus.
- InternetExporter и windows_exporter предоставляют дополнительные метрики о доступности и состоянии системы.
- Grafana доступна на порту 3000, Prometheus — на 9090.
- Subsonic и qBittorrent работают как отдельные сервисы для мультимедиа и загрузок.
- Все сервисы оформлены как службы, что обеспечивает их автозапуск и стабильную работу.
- Docker и WSL на момент диагностики не используются для сервисов мониторинга.

## Использованные команды

```powershell
# Подключение к удалённой машине
Enter-PSSession -ComputerName luigi -Credential luigi\local-admin

# Получить список всех запущенных процессов
Get-Process | Sort-Object ProcessName | Select-Object ProcessName, Id, Path | Format-Table -AutoSize

# Получить список всех запущенных служб
Get-Service | Where-Object {$_.Status -eq "Running"} | Sort-Object DisplayName | Format-Table -AutoSize

# Проверить наличие Docker и его контейнеров
# (если установлен)
docker ps -a

# Проверить наличие WSL-дистрибутивов и процессов
wsl -l -v
wsl -e ps aux

# Поиск процессов Python (экспортеры, скрипты)
Get-Process python* | Select-Object ProcessName, Id, Path | Format-Table -AutoSize

# Поиск процессов Node.js
Get-Process node* | Select-Object ProcessName, Id, Path | Format-Table -AutoSize

# Проверка портов, которые слушают процессы
netstat -ano | findstr LISTEN
```
