## Процедура обновления

### 1. Сборка fullchain.pem

CSR в 2025.05 идентичен CSR из 2025.04 → приватный ключ не менялся.

Собран `fullchain.pem` на локальной машине (PowerShell):
```powershell
Get-Content "...\CER - CRT Files\vkosarev_link.crt", "...\CER - CRT Files\My_CA_Bundle.ca-bundle" `
  | Set-Content "...\vkosarev.link.fullchain.pem" -Encoding ASCII
```
Итоговый файл: `9486 байт` (был `8195 байт`).

### 2. Загрузка на сервер

```powershell
# Загрузка во /tmp (прямая запись в /root/cert/ требует sudo)
pscp -load "vkosarev.link" -i "Z:\MY\vk-amazon-2023-private.ppk" `
  "...\vkosarev.link.fullchain.pem" ubuntu@vkosarev.link:/tmp/fullchain.pem.new

# Установка в /root/cert/
plink ... "sudo cp /tmp/fullchain.pem.new /root/cert/fullchain.pem && \
           sudo cp /tmp/fullchain.pem.new /root/cert/vkosarev.link.fullchain.pem"
```

### 3. Перезагрузка nginx

```bash
sudo docker compose -f /root/vps/vkosarev.link/docker-compose.yml exec nginx nginx -t
# → nginx: configuration file /etc/nginx/nginx.conf syntax is ok

sudo docker compose -f /root/vps/vkosarev.link/docker-compose.yml exec nginx nginx -s reload
# → 2026/05/14 03:25:32 [notice] 28#28: signal process started
```

### 4. Проверка

